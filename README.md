# US 히트맵 (Android 4.2.2 / API 17 대상)

미국 주식시장을 11개 섹터로 나누고, 각 섹터 안의 대표 종목 4개씩(총 44종목)의
실시간 등락을 보여주는 초경량 안드로이드 앱입니다. 구형 이북리더/저사양 기기용.

## 기능
1. **GitHub Actions로 자동 빌드** → 내 PC에 아무것도 설치할 필요 없음
2. **API 키는 GitHub Secret으로 관리** → 소스코드/저장소에 키가 노출되지 않음
3. 화면 항시 켜짐, 상태바 최소화로 화면 전체 활용
4. **화면 전체를 트리맵(Treemap)으로 꽉 채워 44종목을 한 화면에 표시** (스크롤 없음)
5. **박스 크기는 시가총액 비중에 비례** (Finviz류 히트맵과 동일한 방식)
6. 외부 라이브러리 없이 순수 안드로이드 API + 커스텀 Canvas 렌더링 → 매우 가벼움
7. **1시간마다 자동 새로고침** (배터리 절약), 물리 버튼으로 즉시 수동 새로고침 가능
8. 상승=흰 배경/검정 글씨, 하락=검정 배경/흰 글씨로 **흑백 화면에서도 최대 명암 대비**로 구분

## 물리 버튼 동작
- **왼쪽 버튼**: 한국장 ↔ 미국장 전환
- **오른쪽 버튼**: 1시간 기다리지 않고 즉시 최신 정보로 수동 새로고침
- 기기마다 실제 키코드가 달라서 넓게 매핑해뒀습니다. 만약 왼쪽/오른쪽이 반대로 동작하면
  알려주시면 코드에서 바로 바꿔드릴 수 있습니다. 매핑 안 된 키를 누르면 화면에 키코드가
  잠깐 뜨는데, 그 숫자를 알려주시면 정확히 맞출 수 있습니다.
- 버튼을 누르거나 화면을 터치할 때, 그리고 1시간 자동 새로고침 때마다 e-ink 잔상 제거를 위해
  화면이 여러 번 흑백으로 깜빡인 뒤 내용이 표시됩니다 (정상 동작입니다).

## 새로 추가된 기능
- **메인 화면 상단**: 원/달러 환율(exchangerate-api) + 코스피 지수(네이버 금융 비공식 API) + 다우/나스닥/S&P500/VIX(야후 파이낸스, 수치+등락률 함께 표기) 실시간 표시
- **종목 탭 → 상세 화면**: 현재가/등락률, 최근 3일 자체 축적 차트, 당일 시가·고가·저가,
  회사 개요(산업/웹사이트), 재무지표(PER, 52주 최고·최저), 애널리스트 매수/보유/매도 의견
- **자체 3일 차트**: Finnhub 무료 플랜은 과거 차트 데이터(`/stock/candle`)를 제공하지 않아서
  (공식 확인: 무료 플랜은 이 엔드포인트 자체에 접근 불가), 앱이 1분마다 시세를 받아올 때마다
  그 값을 기기 안에 직접 기록해서 차트를 만듭니다. 3일이 지난 기록은 자동으로 삭제됩니다.
  설치 직후에는 점이 거의 없다가, 앱을 계속 켜둘수록 촘촘해집니다.
- **PER**: Finnhub 무료 플랜은 선행(forward) PER을 제공하지 않아 trailing PER로 대체했습니다.
- **코스피 지수**: Finnhub 무료 플랜은 미국 시장 중심이라 한국 지수를 지원하지 않습니다.
  네이버 금융의 비공식 공개 API를 사용했는데, 이건 공식 문서화된 API가 아니라서
  네이버 쪽에서 예고 없이 형식을 바꾸면 이 부분만 "--"로 표시되고 나머지 기능엔 영향 없습니다.

스크롤 없이 화면 전체를 트리맵 형태로 채워서 44종목을 한 화면에 모두 보여줍니다.
각 섹터/종목 박스의 크기는 **시가총액 비중에 비례**합니다 (시가총액이 큰 종목일수록 큰 박스).
시가총액 비중은 자주 바뀌는 값이 아니므로 코드에 근사치를 고정값으로 넣어두었고,
**등락률(%)만 1분마다 실시간 갱신**됩니다 (API 호출량을 늘리지 않기 위한 설계이며,
사각형 배치 계산 자체는 단순 사칙연산이라 기기 성능에 영향을 주지 않습니다).

박스 크기 비율을 바꾸고 싶다면 `MainActivity.java`의 `SECTORS` 배열에서 각 종목 뒤에 있는
숫자(상대적 시가총액 비중, 예: `new TreemapView.Stock("AAPL", 3500)`)만 수정하면 됩니다.

⚠️ **Finnhub 무료 플랜은 분당 60회 호출 제한**이 있습니다. 44종목을 1분마다 조회하면
이 한도에 근접하므로, 가끔 일부 종목이 "오류"로 표시될 수 있습니다. 안정성을 더 높이려면
섹터/종목 수를 줄이거나 새로고침 주기를 늘리면 됩니다 (원하시면 조정해 드릴게요).

## 🚀 GitHub만으로 APK 빌드하기 (프로그램 설치 0개)

### 1단계. GitHub 저장소(repository) 만들기
1. https://github.com 접속 후 로그인
2. 오른쪽 위 `+` → `New repository` → 이름 입력(예: `us-heatmap`) → `Public` → `Create repository`

### 2단계. 이 프로젝트 파일 업로드
1. 저장소 페이지에서 `Add file` → `Upload files`
2. zip 압축 해제한 `HeatmapApp` 폴더 안의 내용물(파일/폴더 전체, `.github` 폴더 포함)을
   브라우저 화면에 드래그 앤 드롭
3. `Commit changes`

### 3단계. Finnhub API 키를 GitHub Secret으로 등록 (코드에 직접 넣지 않음)
1. https://finnhub.io 무료 가입 → API Key 발급
2. 저장소 페이지에서 `Settings` 탭 클릭
3. 왼쪽 메뉴에서 `Secrets and variables` → `Actions` 클릭
4. `New repository secret` 버튼 클릭
5. **Name**: `FINNHUB_API_KEY` (정확히 이 이름이어야 함, 코드/워크플로와 매칭됨)
6. **Secret**: 발급받은 API 키 값 붙여넣기
7. `Add secret` 클릭

이렇게 하면 키가 저장소 파일 어디에도 평문으로 남지 않고, 빌드할 때만 안전하게 주입됩니다.

### 4단계. 자동 빌드 확인 및 APK 다운로드
1. 저장소 상단 `Actions` 탭 클릭
2. `Build APK` 워크플로 실행 확인 (2~4분 소요)
   - 안 보이면 `Actions` → `Build APK` → `Run workflow`로 수동 실행
3. 완료(초록 체크)되면 실행(run) 클릭 → 하단 `Artifacts`에서 `us-heatmap-apk` 다운로드
4. 압축 해제 → `app-debug.apk`가 실제 설치 파일

### 5단계. 기기에 설치
1. `app-debug.apk`를 이메일/USB/클라우드 등으로 이북리더에 전송
2. "출처를 알 수 없는 앱 설치" 허용 후 apk 실행 → 설치

## 네이버 금융 API 프록시 설정 (필수)
구형 안드로이드(4.2.2)는 네이버 금융 API와 직접 통신이 잘 안 되는 문제가 있어서,
**Cloudflare Worker를 통해 우회**합니다 (코스피/국내 종목/다우/나스닥/S&P500/VIX/원달러 전부 해당).
Worker는 네이버 금융, 야후 파이낸스, 환율 API(open.er-api.com) 도메인을 허용하도록 설정되어 있습니다.

1. https://dash.cloudflare.com 무료 가입 (신용카드 불필요)
2. `Workers & Pages` → `Create` → `Create Worker` → 이름 아무거나 → `Deploy`
3. `Edit code` → 기본 코드 전체 삭제 → 저장소의 `naver-proxy-worker.js` 내용 붙여넣기 → `Deploy`
4. 배포 후 나온 주소(`https://이름.계정.workers.dev`) 복사
5. `app/src/main/java/com/example/heatmap/NetUtil.java` 파일에서
   `NAVER_PROXY_BASE` 값을 이 주소로 교체 (끝에 슬래시 `/` 붙이지 않기)
6. `Commit changes` → 자동 재빌드

이 프록시가 없으면 국내장/코스피/다우/나스닥/S&P500/VIX가 전부 "--"로 표시됩니다
(BTC/ETH, 미국 종목은 Finnhub를 직접 쓰므로 이 프록시와 무관하게 정상 작동합니다).

## API 키 보안 관련 참고
GitHub Secret은 **저장소 소스코드에 키가 노출되는 것**을 막아줍니다. 다만 완성된 APK
파일 자체를 디컴파일하면 이론적으로 키를 추출할 수 있습니다 — 이는 클라이언트(모바일) 앱에
API 키를 넣는 모든 방식이 가진 근본적인 한계이며, Finnhub API 특성상 이 이상의 완벽한
은닉은 불가능합니다. 지금 목적(저장소 비공개성 확보)에는 충분한 조치입니다.

## 기타 참고 사항
- 안드로이드 4.2.2는 TLS1.2가 기본 비활성화되어 있어, `TLSSocketFactory` 클래스가 이를 강제 활성화합니다.
- 인터넷(Wi-Fi) 연결이 있어야 실시간 데이터가 갱신됩니다.
- 화면을 계속 켜두므로 배터리 소모가 있습니다.
- 코드 수정 후 `Commit changes`만 누르면 자동으로 재빌드됩니다.
